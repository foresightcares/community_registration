# Documentation Guide

## README.md vs USAGE.md

This project has two main documentation files, each serving a different purpose:

### README.md - Comprehensive Project Documentation
**Purpose**: Complete technical reference and setup guide

**Target Audience**: Developers, system administrators, and technical users

**Contents**:
- Full project overview and architecture
- Installation and environment setup
- Project structure and file organization
- Complete environment variables reference
- Multi-environment configuration (DEV/PRD)
- AWS configuration details
- Cognito integration technical details
- Code examples for integration

**Use When**: 
- Setting up the project for the first time
- Understanding the technical architecture
- Integrating the project into other systems
- Troubleshooting technical issues
- Reference for all configuration options

### USAGE.md - Quick Usage Guide
**Purpose**: Step-by-step instructions for processing registration data

**Target Audience**: End users, operators, and non-technical users

**Contents**:
- Quick setup instructions
- Environment selection guide (DEV/PRD)
- Step-by-step processing guide
- Excel file format requirements
- Example output and results
- Command line options reference
- Common troubleshooting scenarios
- Practical usage examples

**Use When**:
- Running the registration processor
- Preparing Excel files
- Understanding what the script does
- Quick reference for daily operations
- Troubleshooting common usage issues

## Summary

- **README.md** = "How does this work?" (Technical deep dive)
- **USAGE.md** = "How do I use this?" (Practical guide)

Both documents are kept up-to-date and complement each other. Start with README.md for setup, then refer to USAGE.md for daily operations.
